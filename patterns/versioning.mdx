These patterns help you keep pace with the rapid Canton and Splice release cadence and catch breaking changes before they hit your CI pipeline.

## Pre-flight an SDK upgrade

<Note>**Problem.** Splice and Canton ship every one to two weeks. Teams discover breaking changes only after CI breaks.</Note>

**Solution.** Run `get_breaking_changes(from, to)` against the target tags before bumping. It cross-references GitHub release notes with forum activity in the post-release window, so you see both officially announced and community-observed regressions.

**References:**
- [Code & Errors tool](/tools/code-errors)

## Confirm you're on the current Canton + Splice

<Note>**Problem.** Curated knowledge bases and tutorials lag the live release feed by weeks.</Note>

**Solution.** `get_current_versions` returns the latest release tags, cross-referenced with validator-announce mailing list posts that confirm real-world deployment. Always check this before trusting a static tutorial or knowledge base entry.
