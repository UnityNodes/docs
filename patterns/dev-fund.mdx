These patterns help Canton builders avoid duplicate funding requests and improve proposal quality before submission to the Dev Fund.

## Confirm nobody is already building it

<Note>**Problem.** Dev Fund reviewers routinely receive near-duplicate proposals covering LocalNet tooling, validator runbooks, and wallet abstractions.</Note>

**Solution.** Run `detect_builder_overlap(idea)` before drafting. It returns scored matches across existing funded proposals and active ecosystem projects, so you can either differentiate your approach or reach out to collaborate.

## Self-grade a Dev Fund proposal draft

<Note>**Problem.** Approval depends on milestone clarity, named deliverables, and scope realism — easy to miss in a first draft.</Note>

**Solution.** `proposal_success_predictor(draft_text)` walks a ten-point checklist covering milestones, deliverables, referenced sources, document length, and license compliance (Apache 2.0). Aim for a score of ≥ 80% before submitting.
