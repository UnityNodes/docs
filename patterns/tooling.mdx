The Canton and Splice toolchain evolves quickly. Commands that worked six months ago may now point to retired tools, and npm packages that tutorials reference may no longer be maintained. These two patterns give you a repeatable process for catching deprecated tooling before it causes build failures or silent runtime errors in your project.

<Tip>
  Before running any install command suggested by an AI assistant or older tutorial, call the CCPEDIA `check_deprecation` MCP tool. It returns `DEPRECATED`, `CURRENT`, or `NOT_FOUND` for any tool name you pass, along with the recommended replacement.
</Tip>

<AccordionGroup>
  <Accordion title="Verify a CLI tool isn't deprecated before using it">
    **Problem**

    AI coding assistants and older tutorials continue recommending `daml-assistant` and `daml start` long after they were retired in favour of DPM (the Daml Package Manager). Following these suggestions installs or runs tools that no longer work, generating confusing errors and wasting setup time.

    **Solution**

    Always call CCPEDIA's `check_deprecation` MCP tool before running or printing any install command. The tool returns one of three statuses:

    | Status | Meaning |
    |---|---|
    | `CURRENT` | The tool is actively maintained — safe to use |
    | `DEPRECATED` | The tool has been retired — use the replacement listed in the response |
    | `NOT_FOUND` | CCPEDIA has no record of this tool — verify manually |

    ```bash
    # Example MCP call
    check_deprecation({ tool_name: "daml-assistant" })
    # Returns: { status: "DEPRECATED", replacement: "dpm", notes: "Retired in Splice 0.3.x" }
    ```

    Make this call part of any automated scaffolding script or AI agent that generates Canton setup instructions. The tool's deprecation database is updated with each CCPEDIA release, so you always get current data rather than what was true when a tutorial was written.

    **When to use this pattern**

    - Before copy-pasting install commands from blog posts or AI suggestions
    - When writing setup scripts or CI pipelines for a Canton project
    - When onboarding a new team member who may follow old documentation

    **References**
    - [CCPEDIA MCP tool: check_deprecation](/tools/releases-sdks)
  </Accordion>

  <Accordion title="Don't import @daml/ledger or @daml/react">
    **Problem**

    Many tutorials and StackOverflow-style answers still reference `@daml/ledger` and `@daml/react` as the canonical way to interact with the Canton ledger from JavaScript. Both packages are retired. Importing them pulls in stale transport logic that is incompatible with current Canton participant nodes and the CIP-0103 wallet standard.

    **Solution**

    Replace retired packages with the current Canton toolchain:

    | Retired package | Replacement |
    |---|---|
    | `@daml/ledger` | JSON Ledger API v2 + `dpm codegen-js` TypeScript bindings |
    | `@daml/react` | `@canton-network/dapp-sdk` (CIP-0103 wallet transport) |
    | `@daml/types` | Types generated by `dpm codegen-js` |

    ```bash
    # Generate TypeScript bindings for your Daml package
    dpm codegen-js --package my-app --output ./src/generated

    # Install the current dApp SDK
    npm install @canton-network/dapp-sdk
    ```

    Run `list_deprecations` through the CCPEDIA MCP server to scan your `package.json` for any other retired names you may still be referencing. The tool returns a list of all known deprecated Canton npm packages alongside their current replacements.

    ```bash
    # Example MCP call
    list_deprecations({ scope: "npm", filter: "@daml" })
    ```

    **When to use this pattern**

    - When migrating an existing dApp from pre-DPM Canton tooling
    - When reviewing a new project's `package.json` for outdated dependencies
    - Before publishing a tutorial or code example that others will follow

    **References**
    - [CCPEDIA MCP tool: list_deprecations](/tools/releases-sdks)
  </Accordion>
</AccordionGroup>

  Tooling changes are announced in the `validator-announce` mailing list and in GitHub release notes. Use `search_mailing_list({ query: "dpm deprecation", group: "validator-announce" })` via CCPEDIA to find the original announcement for any tool transition you need more context on.
</Note>
