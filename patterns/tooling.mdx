These patterns help you stay current with the Canton toolchain and avoid recommending or depending on packages and CLI tools that have been superseded.

## Verify a CLI tool isn't deprecated before suggesting it

<Note>**Problem.** AI agents keep recommending `daml-assistant` / `daml start` long after they were retired by DPM.</Note>

**Solution.** Always call CCPEDIA's `check_deprecation` MCP tool before printing an install command. It returns `DEPRECATED`, `CURRENT`, or `NOT_FOUND` along with the recommended replacement.

**References:**
- [Releases & SDKs tool](/tools/releases-sdks)

## Don't import @daml/ledger or @daml/react

<Note>**Problem.** Stale tutorials still point to retired npm packages.</Note>

**Solution.** Use the current DA tooling exposed via `dpm codegen-js` for TypeScript bindings, and the `@canton-network/dapp-sdk` for CIP-0103 wallet transport. Run `list_deprecations` to scan for any other package names you might still be using.
