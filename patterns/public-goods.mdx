This pattern describes how the Canton community keeps shared onboarding resources accurate as the network evolves rapidly.

## Keep the Canton onboarding knowledge base current

<Note>**Problem.** The curated Canton onboarding knowledge base drifts behind live releases. The community cannot manually scan every source on every release cycle.</Note>

**Solution.** Use `get_kb_drift` to compare your onboarding documentation against the latest release information. The tool flags entries that reference outdated SDK versions, deprecated tools, or changed configuration values — giving you a prioritized list of corrections to make.
