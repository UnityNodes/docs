Design patterns are small, focused solutions to problems that come up repeatedly when building on Canton. Each pattern here is intentionally concise — you get the problem statement, the recommended solution, the CCPEDIA MCP tools that help you execute it, and pointers to the canonical references. You can read these pages directly, or query any pattern interactively through CCPEDIA's MCP tools such as `semantic_search`, `diagnose_error`, and `get_cip_implementation_status`. The goal is to give you a reliable starting point rather than a comprehensive tutorial, so you can move fast and avoid the common traps that slow Canton teams down.
