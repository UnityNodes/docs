These patterns help Super Validator operators and node runners stay ahead of scheduled network events and navigate the validator onboarding process efficiently.

## Plan around the next MainNet upgrade window

<Note>**Problem.** Ops teams hit traffic freezes, topology freezes, and synchronizer upgrades by surprise.</Note>

**Solution.** Subscribe to the SV calendar via `get_upcoming_operations(days)` or query `get_sv_schedule(environment="MainNet")`. The calendar is refreshed from the SV foundation schedule regularly, so you always see the current planned windows.

## Bootstrap a new validator without re-reading the docs end-to-end

<Note>**Problem.** Validator onboarding has many moving parts: topology, traffic balance, ValidatorLicense, and sequencer configuration.</Note>

**Solution.** Run `get_started_guide(background="enterprise")` for the structured onboarding path, then `get_sv_schedule(environment="DevNet")` to plan around the next DevNet upgrade window. Cross-check with `get_kb_drift()` to ensure any knowledge base entries you consult reflect current release values.
