These patterns help you navigate the Canton Improvement Proposal process, from checking whether a CIP's prerequisites have shipped to following live debates across every channel at once.

## Check whether you can build on a CIP today

<Note>**Problem.** CIPs sit in different states (Draft, In Review, Approved, Final) and may declare `Requires:` blockers that haven't shipped yet.</Note>

**Solution.** Call `detect_cip_dependency_chain(cip_id)`. It returns the full prerequisite chain with each prerequisite's current status. **Final** or **Approved** means done. Anything else still blocks your implementation.

**References:**
- [CIPs tool](/tools/cips)

## Track all conversations about a CIP across surfaces

<Note>**Problem.** A live CIP debate happens on the cip-discuss mailing list, the forum, and GitHub simultaneously; following any one source gives an incomplete picture.</Note>

**Solution.** `get_cip_citations(cip_number)` returns per-source backlinks at per-message granularity. Results include individual mailing list messages, forum posts, GitHub issues and pull requests, blog posts, and recorded talks — all in one view.
