Pick your tool below, follow the integration guide, and you'll have 78 live Canton tools available in your AI assistant in under two minutes.

## Verify the connection

After adding CCPEDIA to your client, it should report **78 tools** available. You can confirm the endpoint is reachable and returning the full tool list from your terminal:

```bash
curl -s -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'
```

A successful response contains roughly 78 entries in the `tools` array. If you see fewer, wait a moment and retry — the server may be mid-sync.

## Try these questions

Once connected, ask any of these in your AI client. Tool routing happens automatically based on the tool descriptions — you do not need to name tools explicitly.

<AccordionGroup>
  <Accordion title="What's the current Canton SDK version?">
    Routes to `get_current_versions`. The tool joins GitHub release tags with posts from the validator-announce mailing list to return the authoritative current version alongside any pending upgrade notices.
  </Accordion>
  <Accordion title="Diagnose this error: TOPOLOGY_TOO_MANY_PENDING_TOPOLOGY_TRANSACTIONS">
    Routes to `diagnose_error`. The tool tokenises the error string and searches Canton Forum threads and GitHub issues for prior fixes, workarounds, and related configuration changes.
  </Accordion>
  <Accordion title="Show dev fund proposals related to validator monitoring">
    Routes to `search_community` and `detect_builder_overlap`. Surfaces existing proposals and active builders in that space so you can avoid duplicating work already in progress.
  </Accordion>
  <Accordion title="What does CIP-0056 require, and is it ready?">
    Routes to `get_cip_implementation_status` and `detect_cip_dependency_chain`. Returns the full readiness graph: which dependencies are merged, which are pending, and which are still draft.
  </Accordion>
  <Accordion title="When is the next MainNet upgrade?">
    Routes to `get_upcoming_operations`. Pulled from the SV calendar, which CCPEDIA refreshes every 12 hours, so you always see the latest scheduled operations window.
  </Accordion>
</AccordionGroup>

## Rate limits

<Note>
  The public endpoint allows **60 calls per 15 minutes per IP address**. If you hit the limit, your quota resets automatically after 15 minutes. For higher throughput, self-host CCPEDIA from the [Apache 2.0 source](https://github.com/UnityNodes/ccpedia) and remove the limit entirely.
</Note>

## Interactive setup wizard

If you prefer a guided flow with auto-detect and one-click copy buttons, open the interactive wizard at [ccpedia.xyz/for-ai](https://ccpedia.xyz/for-ai). It detects your client, generates the correct config snippet, and walks you through pasting it in.
