Before bumping any version in your Canton, Daml, or Splice project you should know what changed, what deprecated, and what is currently live on each network environment. CCPEDIA's 9 release and SDK tools cover all of that: current versions across all packages, per-version changelogs, npm package listings, a full deprecation registry, migration lookup, outdated-guidance detection, and an EVM-to-Canton concept mapper for developers coming from Solidity.

---

### `check_deprecation`

Check whether a specific Canton or Daml toolchain item — CLI, SDK package, or command — is deprecated. Returns `DEPRECATED` (with its replacement), `CURRENT`, or `NOT_FOUND`. Always call this before recommending any Canton tool or package to a developer. The check is scoped to Canton/Daml/Splice only — it is not a general npm or pip deprecation checker.

<Expandable title="Arguments">
  <ParamField path="name" type="string" required>
    Tool, package, or command to check, e.g. `"daml-assistant"`, `"Navigator"`, `"dpm"`, `"canton sandbox"`.
  </ParamField>
</Expandable>

<CodeGroup>
```json MCP call
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "check_deprecation",
    "arguments": { "name": "Navigator" }
  }
}
```
```bash curl
curl -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"check_deprecation","arguments":{"name":"Navigator"}}}'
```
</CodeGroup>

### `get_current_versions`

Get the freshest Canton SDK, Splice, Daml, and DPM versions, plus per-network Splice deployment status (DevNet, TestNet, MainNet) and any upcoming synchronizer-upgrade windows. This data is derived live on every call from two CCPEDIA-only joins: `github_releases` (latest stable tag per repo) and `mailing_messages` (validator-announce list). It is fresher than the Foundation Build-on-Canton KB snapshot, which can lag by weeks. Use this whenever a user asks what version to target or what is live on MainNet.

<Expandable title="Arguments">
  _No arguments. Call with `{}`._
</Expandable>

<CodeGroup>
```json MCP call
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "get_current_versions",
    "arguments": {}
  }
}
```
```bash curl
curl -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"get_current_versions","arguments":{}}}'
```
</CodeGroup>

### `migrate_lookup`

Look up the modern replacement for a deprecated Daml or Canton CLI command, npm package, or tool — for example `"daml start"`, `"@daml/ledger"`, `"Navigator"`, `"Daml Triggers"`, or `"splice-wallet-payments subscription"`. Returns the replacement, the since-version, install instructions where applicable, and a migration note. Use this when a user mentions a specific deprecated symbol; use `list_deprecations` to browse all known deprecation pairs.

<Expandable title="Arguments">
  <ParamField path="query" type="string" required>
    The deprecated command, package, or tool name to look up. Best-match search.
  </ParamField>
</Expandable>

### `outdated_guidance_detector`

Surface CCPEDIA content that mentions deprecated Canton or Daml tooling alongside your query — content that is likely outdated. Cross-references `foundation_kb.DEPRECATED` with doc pages, forum posts, and blog posts. Use this before sharing documentation or forum links to check whether they reference superseded tools or workflows.
