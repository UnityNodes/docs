CCPEDIA's search category gives you 17 tools that reach across every Canton source in the knowledge base — the developer forum, GitHub issues, official documentation, CIPs, governance mailing lists, YouTube transcripts, blog posts, whitepapers, and the ecosystem project registry. Use the narrow, specialised tools when you know exactly what type of content you want; use `full_context` or `semantic_search` when you want an exhaustive cross-source roundup.

---

### `find_apps_by_pattern`

Pattern-search the Featured Apps catalogue by free-text query across name, category, and description. Results are ranked by name and category match first, falling back to description relevance. Use this to discover what apps are live on Canton matching a particular keyword or domain.

<Expandable title="Arguments">
  <ParamField path="pattern" type="string" required>
    Free text, e.g. `"AMM"`, `"wallet"`, `"DeFi vault"`, `"validator"`.
  </ParamField>
  <ParamField path="limit" type="integer">
    Max rows returned (default 10).
  </ParamField>
</Expandable>

<CodeGroup>
```json MCP call
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "find_apps_by_pattern",
    "arguments": { "pattern": "wallet" }
  }
}
```
```bash curl
curl -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"find_apps_by_pattern","arguments":{"pattern":"wallet"}}}'
```
</CodeGroup>

### `search`

Keyword and full-text search over the Canton Network knowledge base — CIPs, Canton/Daml/Splice docs, forum, mailing lists, whitepapers, grant proposals, blog, YouTube, and GitHub. Use this for exact-term or name lookups. For conceptual or "how does X work" questions use `semantic_search` instead. Once you have a doc id from results, call `get_doc` to read the full page.

<Tip>Do not use this tool for other blockchains, the general web, or local files. It is scoped strictly to Canton-ecosystem content.</Tip>

<Expandable title="Arguments">
  <ParamField path="query" type="string" required>
    Search query — an exact term, error code, function name, or phrase.
  </ParamField>
  <ParamField path="type" type="string">
    Optional content-type filter.
  </ParamField>
  <ParamField path="limit" type="integer">
    Max rows returned.
  </ParamField>
</Expandable>

```json MCP call
    "name": "search",
```

### `semantic_search`

Conceptual and meaning-based search over the Canton Network corpus — CIPs, docs, forum, mailing lists, proposals, blog, releases, ecosystem, foundation KB, and YouTube transcripts — using vector plus full-text hybrid retrieval with reranking. Use this for open-ended "how/why/what is" questions or when keyword search returns nothing useful. After getting results, call `get_doc` with a returned id to read the full source page.

<Tip>Do not use this tool for other chains or the general web. For exact-term lookups, `search` is faster and more precise.</Tip>

<Expandable title="Arguments">
  <ParamField path="query" type="string" required>
    Free-form question or phrase, e.g. `"how does Canton handle privacy between counterparties?"`.
  </ParamField>
  <ParamField path="source_types" type="array">
    Restrict results to specific source types. Omit to search all sources.
  </ParamField>
  <ParamField path="limit" type="integer">
    Max rows returned.
  </ParamField>
</Expandable>

<CodeGroup>
```json MCP call
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "semantic_search",
    "arguments": { "query": "how does Canton handle privacy between counterparties?" }
  }
}
```
```bash curl
curl -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \

  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"semantic_search","arguments":{"query":"how does Canton handle privacy between counterparties?"}}}'
```
