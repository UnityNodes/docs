The Canton Dev Fund is the community grant programme that funds builders across the ecosystem. CCPEDIA gives you 5 tools to navigate it intelligently: browse open and closed proposals, understand the funding landscape by category, track milestone issues attached to a specific grant, detect overlap between your idea and existing work, and score a draft proposal against the attributes that correlate with approval. These tools are Canton-ecosystem only and are completely distinct from Canton Improvement Proposals (CIPs), which are governance specifications — not funding requests.

---

### `detect_builder_overlap`

Given a project or proposal idea, find existing Canton ecosystem projects and dev-fund proposals that look similar — across both the Dev Fund proposals corpus and the ecosystem projects catalogue. Use this before submitting a new proposal to cut manual cross-reference research and avoid duplicating work already funded or underway.

<Expandable title="Arguments">
  <ParamField path="idea" type="string" required>
    One-paragraph project idea or capability description, e.g. `"a Canton-native DEX with on-chain orderbook settlement"`.
  </ParamField>
</Expandable>

<CodeGroup>
```json MCP call
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "detect_builder_overlap",
    "arguments": { "idea": "a Canton-native DEX with on-chain orderbook settlement" }
  }
}
```
```bash curl
curl -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"detect_builder_overlap","arguments":{"idea":"a Canton-native DEX with on-chain orderbook settlement"}}}'
```
</CodeGroup>

### `get_funding_landscape`

Get an overview of the Canton Dev Fund: counts of proposals by state and label, top funded categories, and recent activity. Use this to gauge whether a category is over- or under-funded before proposing, and to understand where the community is currently investing its grant budget.

<Expandable title="Arguments">
  <ParamField path="category" type="string">
    Optional label filter, e.g. `"daml-tooling"`, `"node-deployment-operations"`.
  </ParamField>
</Expandable>

<CodeGroup>
```json MCP call

  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "get_funding_landscape",
    "arguments": { "category": "daml-tooling" }
  }
}
```
```bash curl
curl -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"get_funding_landscape","arguments":{"category":"daml-tooling"}}}'
```
</CodeGroup>

### `get_proposal_milestones`

For one Canton Dev Fund proposal PR, list any related milestone tracking issues — issues whose body or title references the PR number. Use this to follow the execution progress of an approved grant and understand which deliverables have been completed or are still open.

<Expandable title="Arguments">

    PR number in the `canton-foundation/canton-dev-fund` repository.
  </ParamField>
</Expandable>


### `list_proposals`

List Canton Network Dev Fund grant proposals — community funding requests tracked on GitHub — filterable by state (`open`, `closed`, or `all`). Use this for "what grants and funding requests exist?" questions. Returns number, title, state, author, and board status. Note that these are funding requests, not governance specifications — for CIPs use `list_cips` and `get_cip`.

<Expandable title="Arguments">
  <ParamField path="state" type="string">
    Filter by state: `"open"`, `"closed"`, or `"all"`.
  </ParamField>
  <ParamField path="limit" type="integer">
    Max rows returned.
  </ParamField>
</Expandable>

<CodeGroup>
```json MCP call
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "list_proposals",
    "arguments": { "state": "open" }
  }
}
```
```bash curl
curl -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"list_proposals","arguments":{"state":"open"}}}'
```
</CodeGroup>

### `proposal_success_predictor`

Get a heuristic readiness score for a Dev Fund proposal draft, based on attributes that correlate with approval in the historical corpus: explicit milestones, code or proof-of-concept references, cited sources, and scope realism. This is not a vote — it is a structured checklist that helps you identify gaps before submitting.

<Expandable title="Arguments">
  <ParamField path="draft_text" type="string" required>
    Full draft text of the proposal.
  </ParamField>
</Expandable>

<CodeGroup>
```json MCP call
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "proposal_success_predictor",
    "arguments": { "draft_text": "## Summary\nWe propose to build..." }
  }
}
```
```bash curl
curl -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"proposal_success_predictor","arguments":{"draft_text":"## Summary\nWe propose to build..."}}}'
```
